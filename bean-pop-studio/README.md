# Bean Pop Studio

把照片快速转成可开工的拼豆图纸：上传图片后生成成品预览、制作图纸、色号清单和颗数统计，并支持隐藏底色、合并颜色、矩形改豆点和图纸下载。

## 项目定位

Bean Pop Studio 面向第一次做拼豆、想做头像/宠物/礼物图的用户。产品目标不是做复杂的专业像素编辑器，而是让用户用最短路径完成：

1. 选照片
2. 看成品
3. 拿图纸、色号和颗数
4. 必要时再微调尺寸、颜色、品牌和局部豆点

当前版本是浏览器端 MVP，所有图片处理、图纸生成和导出都在本地完成，不需要后端服务。

## 技术栈

- Vue 3 Composition API：页面状态、制作弹窗、参数联动和交互组件。
- Vite 5：开发服务与生产构建。
- Canvas 2D：图片预处理、拼豆成品预览、制作图纸、下载图纸渲染。
- Vitest：核心算法、图片预处理、状态和材料估算测试。
- LocalStorage：本地作品保存能力。

## 核心实现

### 1. 制作流程

`App.vue` 统一管理制作状态，包括原图、预处理图、参数配置、结果矩阵、色号统计、隐藏颜色、编辑历史和当前画布模式。`GenerationDialog.vue` 承载三步流程：

- 上传照片：支持选择图片和拖拽上传。
- 调整参数：设置成品宽度、最大颜色数、豆子品牌、画面比例、取景方式和风格处理。
- 查看结果：默认看成品，可切换原照片、取景、成品感、制作图纸。

渲染通过 `requestAnimationFrame` 合并调度，窗口或容器尺寸变化时重新绘制 Canvas。

### 2. 图片预处理

`imagePrep.js` 负责让普通照片更适合转拼豆：

- 透明图优先用 alpha 边界识别主体。
- 普通照片用边缘平均色和边缘噪声估算主体区域。
- 支持原图比例、1:1、4:5、3:4、3:2 等比例。
- 支持留白、平衡、靠近主体三种取景。
- 提供原图直转、柔和卡通、潮玩贴纸、清爽描边等风格方向。

### 3. 拼豆算法

核心在 `beadEngine.js`：

- 将品牌色盘从 HEX 转 RGB，再转 CIE Lab。
- 使用 CIEDE2000 色差计算，把图片像素映射到真实拼豆色号。
- 按目标最大边长等比例生成网格，例如 `32 × 42` 或更大尺寸。
- 对每个网格单元采样并生成细节分数，用于区分边缘、轮廓和低信息背景。
- 根据候选色使用频率、权重、暗色锚点、亮色锚点和高饱和锚点选择活跃色盘。
- 颜色数较高时使用 4×4 Bayer 矩阵做轻量抖动，减少大块渐变断层。
- 颜色数较低时做孤立点平滑，减少噪点。
- 统计每个色号的颗数，并按外圈优先排序，方便用户从边缘色开始处理。

### 4. 图纸与编辑

结果态支持：

- 底色隐藏：从边缘 flood-fill 找到背景豆点，隐藏后同步更新颗数。
- 色号隐藏/显示：点击色号可隐藏对应颜色，再次恢复。
- 合并颜色：在色号列表中多选相近颜色，默认保留颗数更多的主色。
- 改豆点：选择右侧色号，在画布中拖拽矩形区域批量改色。
- 撤销：保存最近编辑快照，支持回退。
- 制作图纸视图：`canvas-sheet` 会基于有效拼豆内容裁切并放大，让核心网格尽量铺满预览区域。
- 下载图纸：导出包含图纸、色号和颗数的 PNG。

## 产品差异化

- 更快：默认示例可直接跑完整流程，用户不上传也能理解结果。
- 更轻：不在首页堆复杂参数，核心功能集中到制作弹窗。
- 更像产品：先展示成品感，再提供图纸、色号和颗数。
- 更好改：合并颜色、隐藏底色、矩形改豆点放在结果页直接操作。
- 更适合中文用户：文案围绕礼物、宠物、头像和开拼场景，不是单纯工具参数。

## 当前限制

- 色盘仍是内置静态数据，真实品牌全量色号需要继续补齐和校准。
- 主体识别为启发式算法，不是机器学习抠图；复杂照片仍可能需要用户手动改豆点。
- 当前导出以 PNG 为主，PDF、多页板位和打印比例仍待完善。
- 本地作品保存依赖浏览器 LocalStorage，暂不支持账号同步。

## 后续路线

- 补齐 Perler/Hama/Artkal/MARD 全量标准色盘，并标注停产/难买色。
- 增加 PDF 导出：按 29×29 或 58×58 拼板分页。
- 增加购物清单：按色号统计袋数、余量和缺色提醒。
- 增加移动端逐板模式：一块板一块板查看，已完成豆点可标记。
- 增加更强的主体提取：接入前景分割或用户手动画框。
- 增加导入/导出项目 JSON，支持二次编辑和分享。
